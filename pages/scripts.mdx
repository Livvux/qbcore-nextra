## Getting Started with Scripts

### Installation Steps

1. **Download the Script**: Get the script files from trusted sources
2. **Extract Files**: Place the script folder in your server's `resources` directory
3. **Install Dependencies**: Ensure all required dependencies are installed
4. **Update server.cfg**: Add `start [script_name]` to your configuration
5. **Configure Settings**: Modify config files as needed for your server
6. **Restart Server**: Launch your server and test the new functionality

### Best Practices

- **Test Thoroughly**: Always test scripts on a development server first
- **Read Documentation**: Review all available documentation before installation
- **Check Dependencies**: Ensure all required resources are properly installed
- **Backup Your Server**: Create backups before installing new scripts
- **Monitor Performance**: Watch for any impact on server performance

## Script Development

### Creating Your Own Scripts

Want to create custom scripts for QBCore? Here are some resources:

- [QBCore Development Guide](/tutorials/scripting)
- [API Documentation](/docs/core)
- [Event Reference](/docs/core/events)
- [Function Library](/docs/core/functions)

### Common Script Patterns

```lua
-- Basic QBCore script structure
local QBCore = exports['qb-core']:GetCoreObject()

-- Client-side player loaded event
RegisterNetEvent('QBCore:Client:OnPlayerLoaded', function()
    -- Player loaded logic here
end)

-- Server-side callback example
QBCore.Functions.CreateCallback('script:getData', function(source, cb)
    local Player = QBCore.Functions.GetPlayer(source)
    cb(Player.PlayerData)
end)
```
